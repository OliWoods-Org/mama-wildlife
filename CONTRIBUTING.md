# Contributing

Thank you for your interest in contributing to this OliWoods Foundation project. Every contribution helps us build AI tools that serve people who need them most.

## How to Contribute

### Report Issues
- Use [GitHub Issues](../../issues) to report bugs or suggest features
- Include as much detail as possible

### Submit Code
1. Fork the repository
2. Create a feature branch (`git checkout -b feat/your-feature`)
3. Make your changes
4. Write or update tests if applicable
5. Commit with a clear message (`git commit -m "feat: add your feature"`)
6. Push to your fork (`git push origin feat/your-feature`)
7. Open a Pull Request

### Other Ways to Help
- **Domain expertise** — review agent logic, suggest improvements
- **Documentation** — improve README, add guides, fix typos
- **Translation** — help make tools accessible in more languages
- **Testing** — try the tools and report what works
- **Spread the word** — share with people who could benefit

## Code of Conduct

- Be kind, respectful, and constructive
- Remember the people these tools serve are in vulnerable situations
- Prioritize safety over features
- No tolerance for discrimination of any kind

## Safety Guidelines

Many of these projects serve people in crisis:

- **Never remove or weaken crisis handoff mechanisms** (988, SAMHSA, etc.)
- **Never store PII** beyond what is absolutely necessary
- **Always err on the side of caution** — when in doubt, refer to a human
- **Test safety features** more rigorously than any other code

## Questions?

Open an issue or reach out via the [OliWoods Foundation](https://oliwoodsfoundation.org).
